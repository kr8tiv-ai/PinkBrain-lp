## 1. Bags API Research

### Authentication
- Uses `x-api-key` header for authentication
- API keys obtained from Bags.fm developer portal
- Session-based auth (Bags Agent) uses challenge-response flow with 15-minute TTL

### Rate Limiting
- **Limit:** 1,000 requests per hour per user + IP combination
- **Headers:** `X-RateLimit-Remaining`, `X-RateLimit-Reset`
- **Strategy:** Token bucket algorithm with reserved capacity for critical operations

### Key Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/fee-share/config` | GET | Retrieve fee routing configuration |
| `/fee-share/wallet/bulk` | POST | Batch query wallet balances |
| `/partner/claim` | POST | Generate claim transaction for partner fees |
| `/token-launch/claim-txs/v2` | POST | Claim token position fees |
| `/trade/quote` | GET | Get swap quote |
| `/trade/swap` | POST | Generate swap transaction |

### Transaction Flow
1. All transactions generated by Bags API are serialized Solana transactions
2. Transactions include proper Program IDs for fee routing
3. Bags platform takes its cut within the claim transaction itself
4. **Critical:** Never bypass Bags fee routing - use their endpoints only

## 2. Meteora DAMM v2 Research

### Why DAMM v2 (Not DLMM)
Per Meteora documentation:
> "DLMM pools don't provide LP tokens upon adding liquidity, so once the pool is created, liquidity deposited cannot be locked permanently (unlike dynamic AMM pools)."

DAMM v2 supports:
- Position NFTs representing LP ownership
- Permanent locking while preserving fee claim capability
- Concentrated liquidity via price range
- Dynamic fees

### Program ID
- **Mainnet & Devnet:** `cpamdpZCGKUy5JxQXB4dcpGPiikHawvSWAd6mEn1sGG`

### SDK Installation
```bash
npm install @meteora-ag/cp-amm-sdk
```

### Key SDK Functions

```typescript
import { CpAmm } from '@meteora-ag/cp-amm-sdk';

// Initialize
const cpAmm = new CpAmm(connection);

// Pool operations
await cpAmm.getAllPools();
await cpAmm.fetchPoolState(poolAddress);
await cpAmm.createPool(/* params */);
await cpAmm.createCustomPool(/* params */);

// Position operations
await cpAmm.createPosition(/* params */);
await cpAmm.addLiquidity(/* params */);
await cpAmm.permanentLockPosition(/* params */); // IRREVERSIBLE
await cpAmm.claimPositionFee(/* params */); // Works on locked positions
```

### Pool Parameters
- Token A mint, Token B mint
- Base fee (percentage per swap)
- Initial sqrt price (from current market price)
- Price range (min/max for concentrated liquidity)
- Fee scheduler mode (optional)
- Dynamic fee toggle (optional)

### Permanent Lock Behavior
- **Irreversible** - Once locked, base assets cannot be withdrawn
- Fee claiming still works on locked positions
- Provides cryptographic proof to community that liquidity is secured

## 4. Solana Transaction Building

### Compute Budget for Priority Fees
```typescript
import { ComputeBudgetProgram } from '@solana/web3.js';

// Add to transaction before signing
const modifyComputeUnits = ComputeBudgetProgram.setComputeUnitLimit({
  units: 200_000
});

const addPriorityFee = ComputeBudgetProgram.setComputeUnitPrice({
  microLamports: priorityFeeEstimate
});

transaction.add(modifyComputeUnits);
transaction.add(addPriorityFee);
```

### Address Lookup Tables (LUTs)
For batched distribution transactions:
```typescript
// Compress address references in large transactions
const lookupTableAccount = await connection.getAddressLookupTable(lutAddress);
```

## 6. Gotchas & Edge Cases

### Bags API
- Session expires in 15 minutes - refresh before critical operations
- Rate limit is per user+IP, not per API key
- Claim transactions must be signed before submission

### Meteora DAMM v2
- Permanent lock is IRREVERSIBLE - no undo
- Pool creation requires initial liquidity
- Fee claiming from locked positions works but requires specific function call

### Helius
- Priority fees are estimates - may need adjustment during congestion
- DAS API has different rate limits than RPC
- Webhook endpoints must be publicly accessible

### Solana
- Recent blockhash expires in ~60-90 seconds
- Transaction size limit: 1232 bytes (without LUTs)
- Commitment levels: `processed`, `confirmed`, `finalized`
